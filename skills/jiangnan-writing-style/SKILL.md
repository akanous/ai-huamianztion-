# 江南写作风

> 写给想写出江南那种"少年感"文字的人。
> 也写给不想让自己的文字一眼被认出是AI的人。

本技能融合了三件事：

1. **江南叙事协议**——画面先行、缺失驱动、反差构图、信物系统、不完整闭合
2. **去AI痕迹（humanizer）**——29种AI写作模式的识别与修正
3. **交付规范**——简体中文 + 宋体 + Word文档输出

---

## 写作总流程

```text
用户需求 → Step 0 心理锚定
         → Step 1 配方提取（角色/信物/画面）
         → Step 2 技法引擎（写稿）
         → Step 3 写作规范执行（简体/宋体/去AI）
         → Step 4 自我检视（扫描AI痕迹）
         → Step 5 输出交付（生成docx）
         → Step 6 质检清单
```

## Step 0: 心理锚定与创作伦理

### 石匠心态

我们不是在"代写"。我们是在帮一个创作者找到属于他自己的句号。

### 创作伦理铁律

- **E1 缺失驱动**：故事由"失去什么"驱动，不是由"获得什么"驱动
- **E2 不跪下去**：拒绝爽文逻辑，保留少年式固执——主角可以不赢，但不能跪
- **E3 修订伦理学**：不能用今天的成熟去否认昨天的孤独——角色的成长要尊重他过去的每一次跌倒
- **E4 温暖底色**：再深的孤独也要留最后一道光，不是强行HE，是让读者觉得"活着还是值得的"
- **E5 意象化回答**：对未解之谜不给确定答案，只给符合情感的意象——意象>逻辑

## Step 1: 写作前准备

### 结局尺度选择

| 模式 | 字数 | 用法 |
|------|------|------|
| Fragment（片段） | 200~800字 | 电影彩蛋、一个瞬间的画面、情感余韵 |
| Grand Finale（完整章节） | 2000~10000字 | 多线索交汇、多重压力释放、完整叙事 |

### 三元素提取（每次动笔前先回答）

1. **衰小孩 & 代价**：「谁在这个故事里？他/她为了什么不能放弃的理由，付出了什么代价？」
2. **情感信物**：「有没有一个具体的东西承载这段感情？」（饭团、弹珠汽水、旧笔记本等）
3. **第一画面**：「闭上眼，你脑海中那个最想看到的瞬间是什么样的？」（必须是一个具体的画面，不是一种情绪）

## Step 2: 技法引擎

### Fragment模式：时间折叠 + 极致留白

必用句式结构：
- **时间折叠起手**——"很多年以后，他还会想起那个下午"
- **物哀定格**——在关键时刻停下来描写一个无关紧要的小物件
- **未完成动作**——手伸到一半停住了、话没说完就被风吹散了
- **软转折收尾**——不总结，不含蓄升华，只用画面收束

### Grand Finale模式：五幕结构 + 多重施压

```text
平凡日常 → 压力累积 → 假平静 → 爆发 → 余韵
```

**核心技法：**

**反差构图**——同一个画面里放两个极端：热血和日常、巨大和渺小、战斗和饭团。
例如：指尖能射出贯穿怪物的光束，却不知道该怎么把饭团自然地递给别人。

**信物系统**——找一个具体的东西贯穿叙事。它不需要贵重，但每次出现都有不同的重量。
例如：便利店的饭团。第一次出现是"不知道怎么表达关心"，第二次是"熟悉的安全感"，第三次可能是"再也回不去的日常"。

**不完整闭合**——结局不把每一根线都系紧。留一根线松松地垂着，让读者自己去揣摩。
例如：打完最终boss后不去拥抱庆祝，而是坐在废墟上分一罐温热的弹珠汽水。

## Step 3: 写作规范

### 3a. 语言文字规范（强约束）

- **全文使用简体中文**（GB2312/GBK字符集）
- **不得出现日文汉字**：霈/霊→灵、圧→压、鷹→鹰、歳→岁、熒→荧
- **不得出现繁体字**（如「靈壓」→「灵压」）
- **删除日文假名注音**（如 あさば つかさ 全部清除）
- **专有名词中的日文汉字需转简体**
- **中文标点使用全角**（，。！？——……）

### 3b. 句式规范（防AI痕迹）

- **破折号使用上限**：一章不超过3个「——」。能用句号断句就别用破折号
- **比喻节制**：同一段不超过1个「像……（一样）」。保留最有力的那个
- **「不是……是……」句型上限**：全篇不超过2次
- **对话标签精简**：上下文能看出谁在说话就不加「他说」「她说」
- **虚词控制**：删掉多余的「大约」「像是」「仿佛」「似乎」
- **节奏变化**：强制插入1-2个极短段落（1-3字成段）打破均匀节奏
- **「然后」「接着」等连接词**：一章不超过3次，场景切换用留白代替过渡词

### 3c. 字体规范

- 所有中文正文使用**宋体（SimSun）**，字号10.5pt（五号）
- 不使用游明朝（Yu Mincho）、MS Mincho等日文字体
- 标题：宋体18pt加粗居中
- 行距1.2倍，首行缩进0.75cm

## Step 4: 自我检视（去AI痕迹扫描）

完成初稿后，逐项扫描以下AI痕迹。这是**必须执行**的步骤，不可跳过。

### AI模式中文自查表

请逐条对照初稿，如有发现立即修正：

| 编号 | AI痕迹 | 修复方法 |
|------|--------|----------|
| 1 | 破折号超过3个 | 换成句号或自然停顿 |
| 2 | 比喻堆叠（同段2+个「像」） | 砍到1个 |
| 3 | 「不是……是……」重复出现 | 只留1处 |
| 4 | 句式匀速无变化 | 插短句 |
| 5 | 对话标签过密 | 去掉一半 |
| 6 | 「大约」「像是」「仿佛」过量 | 删大部分 |
| 7 | 「然后」「接着」连接过多 | 用留白代替 |
| 8 | 段落长度过于均匀（全是2-4行） | 穿插1行长段和1行短段 |
| 9 | 每段都做场景铺垫再进入核心 | 砍掉铺垫句，直接进画面 |
| 10 | 结尾做总结升华 | 砍掉总结，用画面收尾 |

### 深层AI痕迹

| 模式 | 中文对应 | 说明 |
|------|----------|------|
| 过度强调重要性 | 「标志着一个关键时刻」「证明了……的深远意义」 | 直接陈述事实，不要赋予意义 |
| 空洞的副词修饰 | 「深刻地」「充分地」「有力地」 | 删掉 |
| "-ing"假深度 | 「凸显了……」「反映出了……」「体现了……」 | 换成主谓宾短句 |
| "Not only...but also" | 「不仅……更……」 | 全篇不超过1次 |
| 规则三分法 | 任何自然出现的三元素并列 | 如果感觉到工整，打散它 |
| 同义词替换 | 同一段内换不同词说同一个事 | 重复用同一个词，更自然 |
| 模糊归因 | 「业内专家指出」「有观点认为」 | 要么给具体出处，要么删掉 |
| 客服式语气 | 「希望这对你有帮助」「请随时告诉我」 | 全部删除 |
| 虚张声势的深度 | 「问题的核心在于」「真正重要的是」 | 直接说想说的话，不用前置渲染 |
| 口号式结尾 | 「未来充满希望」「一个崭新的篇章正在开启」 | 换成具体的画面或动作 |

## Step 5: 输出交付（Word文档）

### 前置准备

```bash
pip install python-docx
```

### Python生成脚本模板

```python
from docx import Document
from docx.shared import Pt, Cm
from docx.enum.text import WD_ALIGN_PARAGRAPH
from docx.oxml.ns import qn

doc = Document()

# 页面设置（A5文库本尺寸）
section = doc.sections[0]
section.page_width = Cm(14.8)
section.page_height = Cm(21.0)
section.top_margin = Cm(2.5)
section.bottom_margin = Cm(2.5)
section.left_margin = Cm(2.8)
section.right_margin = Cm(2.8)

# 全局字体：宋体 + 简体中文
style = doc.styles['Normal']
style.font.name = '宋体'
style.font.size = Pt(11)
style.paragraph_format.line_spacing = 1.2
style.paragraph_format.space_after = Pt(6)
rpr = style.element.rPr
rFonts = rpr.find(qn('w:rFonts'))
if rFonts is None:
    rFonts = style.element.rPr.makeelement(qn('w:rFonts'), {})
    rpr.append(rFonts)
rFonts.set(qn('w:eastAsia'), '宋体')
rFonts.set(qn('w:ascii'), '宋体')
rFonts.set(qn('w:hAnsi'), '宋体')

# 标题
title = doc.add_paragraph()
title.alignment = WD_ALIGN_PARAGRAPH.CENTER
run = title.add_run('第一章  灵压')
run.font.size = Pt(18)
run.font.bold = True
run.font.name = '宋体'
run.element.rPr.rFonts.set(qn('w:eastAsia'), '宋体')
run.element.rPr.rFonts.set(qn('w:ascii'), '宋体')
run.element.rPr.rFonts.set(qn('w:hAnsi'), '宋体')
title.paragraph_format.space_after = Pt(20)

# 段落辅助函数
def add_para(text, indent=True):
    p = doc.add_paragraph()
    p.paragraph_format.first_line_indent = Cm(0.75) if indent else Cm(0)
    p.paragraph_format.line_spacing = 1.2
    p.paragraph_format.space_after = Pt(4)
    run = p.add_run(text)
    run.font.size = Pt(10.5)
    run.font.name = '宋体'
    run.element.rPr.rFonts.set(qn('w:eastAsia'), '宋体')
    run.element.rPr.rFonts.set(qn('w:ascii'), '宋体')
    run.element.rPr.rFonts.set(qn('w:hAnsi'), '宋体')
    return p

# 正文（将章节文本逐段加入）
texts = [
    "第一章的正文段落……",
    "每个字符串是一个段落。",
]
for t in texts:
    add_para(t)

# 保存
doc.save(r"C:\Users\Admin\Desktop\jn\第一章_灵压.docx")
print("文档已保存")
```

## Step 6: 质检清单

交付前逐项自检：

### 叙事技法自检
- [ ] **画面先行**：开篇第一句是一个具体画面，不是背景介绍
- [ ] **缺失驱动**：驱动情节的是失去了什么，不是想得到什么
- [ ] **反差构图**：叙事中有没有把宏大和日常放在一起的画面？
- [ ] **信物在场**：有没有一个具体物件贯穿情感线？
- [ ] **软转折密度**：场景转换是否自然，没有生硬的切割？
- [ ] **不完整闭合**：结尾有没有留一根线没系紧？
- [ ] **无直接情绪词**：没有「开心」「难过」「感动」——用画面让读者自己感受

### AI痕迹自检
- [ ] **无破折号过度**（≤3个/章）
- [ ] **无不必要比喻堆叠**
- [ ] **无双重否定/「不是……是……」句式超2次**
- [ ] **有节奏变化**：有1-2字极短段打破匀速
- [ ] **对话标签精简**：不超过需要的一半
- [ ] **无虚词填充**：「大约」「像是」「仿佛」「似乎」均已被清理
- [ ] **无口号式结尾**：不用总结升华
- [ ] **无客服式语气**：无「希望对你有帮助」「请告诉我」

### 输出规范自检
- [ ] **字体为宋体**
- [ ] **全文简体中文**
- [ ] **汉字字数达标**
- [ ] **无日文汉字/假名残留**
- [ ] **文件路径正确**

## 诚实边界

✅ 擅长：短篇-中篇情感叙事（100~10000字）、人物出场/情感戏/告别场景、润色已有文字、去AI痕迹

⚠️ 不擅长：长篇架构（>2万字）、纯喜剧、学术/公文/新闻、女性角色第一人称
