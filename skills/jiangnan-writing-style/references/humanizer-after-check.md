# 交付后检查：AI痕迹扫描速查表

完成一章后，按 jiangnan-writing-style 的 Step 4 做一次 AI 痕迹检视。

## 中文写作中常见 AI 痕迹

| 模式 | 表现形式 | 修改方向 |
|------|----------|----------|
| 白描不足 / 比喻堆叠 | 同一段用 2-3 个「像……一样」 | 能用白描解决的问题不用比喻；同段不超过1个「像」。减少比喻本身也是去AI痕迹 |
| 破折号过度 | 一章出现 8+ 次叙事破折号 | 换成句号断句或自然停顿，保留 ≤3 处。对话内破折号（「那个——」）不计入 |
| 「不是……是……」对称 | 反复用「不是X，是Y」句型 | 只留一处做节奏变化，其余改直接陈述 |
| 叙事短句独立成段过多 | 「眼睛是蓝色的。」「但他不会认错。」单独成段 | 合并到上下文段落中，全章极短叙事段不超过2处，仅用于刻意节奏冲击 |
| 对话标签过多 | 每个对话都带「他说」「她说」 | 去掉一半，上下文能看出谁在说就不加标签 |
| 虚词填充 | 「大约」「像是」「仿佛」反复出现 | 全部删掉，用具体数字或直接描述替代 |
| 「然后」「接着」过度 | 场景转换靠连接词衔接 | 用留白和自然过渡代替，全章不超过3次 |
| 分段过于均匀 | 所有段落长度相近 | 穿插不同长度的段落 |
| 每段带铺垫 | 先写环境再进核心 | 砍掉第一句铺垫句，直接进画面 |
| 结尾升华 | 用总结性句子收尾 | 换成具体画面或动作，不要总结 |

## 极短段使用原则

- 全章极短叙事段（1-3汉字）不超过 **2处**
- 仅用于确实需要一击式冲击的时刻（怪物登场、重大转折、核心台词）
- 单纯陈述句、过渡句、可以自然合并的短句 → 一律合并到相邻段落
- 对话中的简短回应（"嗯。" "结界呢？"）不受此限，它们是正常对话节奏

## 白描优先原则

- 能用直接描述解决的问题不要用比喻
- 写「空气像变稠了，贴在皮肤上有点黏」优于「像暴风雨到来之前那种沉闷的低气压」
- 「像是」「仿佛」「似乎」等虚拟词不构成有效比喻，直接删除或用实词替代
- 叙事段优先使用：具体动作、环境细节、感官描写（光、声、气味、温度、触感）
- 比喻只在对读者理解至关重要的地方使用（如怪物首次登场的视觉冲击）

## 流程

写作（jiangnan-writing-style）→ docx 输出 → AI痕迹扫描 → 修改 → 最终交付
